* 1. 자발적으로 최고 수준의 목표를 세우고 끈질기게 성취한 경험에 대해 서술해 주십시오. (본인이 설정한 목표 / 목표의 수립과정 / 처음에 생각했던 목표달성 가능성 / 수행 과정에서 부딪힌 장애물 및 그 때의 감정(생각) / 목표 달성을 위한 구체적 노력 / 실제 결과 / 경험의 진실성을 증명할 수 있는 근거가 잘 드러나도록 기술) (700~1000자 10단락 이내) 

전자공학과 졸업 후, 기존 하드웨어 지식에 더해 6개월 내 데이터 분석 전문가 수준의 역량을 갖춰 생산 최적화 분야의 차별화된 인재가 되겠다는 목표를 세웠습니다. 반도체 FinFET 공정 최적화 프로젝트를 진행하며 온도 1000°C→800°C 감소, 물질량 2.96μm→2.08μm 절약, 공정 횟수 5회 감소라는 성과를 얻으면서 생산 현장에서 데이터 분석 역량이 핵심 경쟁력이 될 것이라 확신했기 때문입니다.

솔직히 처음엔 목표 달성 가능성을 50% 정도로 생각했습니다. 전자공학 전공자가 6개월 만에 BigQuery, SQL, Python을 마스터하고 실무 프로젝트까지 완성한다는 것은 분명 도전적이었습니다. 가장 큰 장벽은 170만 건 배달앱 데이터 분석 프로젝트에서 마주했습니다. 52,823명 사용자의 25주간 행동 패턴을 분석하며 56.78%라는 높은 휴면율을 발견했지만, 초기에는 단순 통계에만 머물러 '이게 정말 의미 있는 분석일까?'라는 회의감이 들었습니다.

저는 매일 8시간씩 집중 학습하며 데이터 퍼널 분석과 리텐션 분석을 통해 Screen View(100%)→Payment(21.71%) 극심한 이탈 원인을 규명하고, 유저별 타입별로 분류하여 휴면 유저 결제율(18.65%)이 신규 유저(13.38%)보다 높다는 인사이트를 발견했습니다. "휴면 유저도 결제 의향이 있다"는 가설을 데이터로 검증한 순간이었으며, 6개월 만에 BigQuery에서 복잡한 SQL 쿼리를 구사하며 1주차 리텐션 8%→20% 개선, 휴면율 57%→40% 감소라는 구체적 KPI 목표와 6가지 실행 방안을 제시했습니다. 취득한 3개 자격증과 실제 배달앱 데이터 분석 코드와 리포트가 객관적 증거입니다. 이 경험을 통해 MES 데이터 기반 생산실적 분석, Lead Time 최적화, 공정 가동율 개선 등 귀사 생산관리/생산기술 업무에 즉시 기여할 수 있는 역량을 갖췄다고 확신합니다.


* 2. 새로운 것을 접목하거나 남다른 아이디어를 통해 문제를 개선했던 경험에 대해 서술해 주십시오. (기존 방식과 본인이 시도한 방식의 차이 / 새로운 시도를 하게 된 계기 / 새로운 시도를 했을 때의 주변 반응 / 새로운 시도를 위해 감수해야 했던 점 / 구체적인 실행 과정 및 결과 / 경험의 진실성을 증명할 수 있는 근거가 잘 드러나도록 기술) (700~1000자 10단락 이내)
스마트워치 센서 행동 분류 프로젝트에서 기존 방식은 단순히 전체 정확도 97.5%에 만족하는 것이었습니다. 하지만 저는 실제 현장 활용 가능성을 고려한 클래스별 성능 분석이라는 새로운 접근을 시도했습니다. 6가지 행동 패턴(서기, 앉기, 눕기, 걷기, 계단 오르내리기) 분류에서 높은 정확도를 얻었지만, 실제 현장에서는 특정 행동의 오분류가 치명적 문제가 될 수 있다는 생각이 들었습니다. 특히 'LAYING' 1,115개 vs '계단 내려가기' 791개의 클래스 불균형 문제를 발견했습니다.

동료들은 "이미 97.5% 정확도면 충분하지 않나?"라며 의아해했습니다. 추가 분석에 소요되는 시간과 SMOTE 기법 학습이라는 리스크를 감수해야 했지만, 실무 적용 가능한 견고한 모델을 만들겠다는 신념으로 진행했습니다. 562개 센서 특성을 체계적으로 분석하며 어떤 센서 그룹이 각 행동 분류에 중요한지 파악했습니다. SMOTE기법을 도입하여 부족한 클래스의 합성 데이터를 생성했으며, 단순 복제가 아닌  KNN(K-Nearest Neighbors)을 활용한 새로운 합성 샘플 생성으로 클래스 균형을 맞췄습니다.

또한 모델을 구현할 때, Random Forest,  xgboost 알고리즘 등 다양한 기법을 실험하며 데이터 특성에 최적화된 방법을 찾았고, 변수별 중요도를 파악하고 해당 변수에 대한 가중치를 두었습니다. 너무 많은 변수들은 학습에 부정적 영향을 미칠 수 있어, 562개의 변수 중 중요도 상위 50, 100, 150개를 선별하여 최적의 파라미터를 찾고 모델링에 적용했습니다. 이 결과들은 행렬 분석 결과 등을 통해 적용 전후의 성능을 비교했으며, 기존 97%였던 정확도를 99%까지 끌어올릴 수 있었습니다.

3. 지원분야와 관련하여 특정 영역의 전문성을 키우기 위해 꾸준히 노력한 경험에 대해 서술해 주십시오. (전문성의 구체적 영역 (예. 통계분석) / 전문성을 높이기 위한 학습 과정 / 전문성 획득을 위해 투입한 시간 및 방법 / 습득한 지식 및 기술을 실전적으로 적용해 본 사례 / 전문성을 객관적으로 확인한 경험 / 전문성 향상을 위해 교류하고 있는 네트워크 / 경험의 진실성을 증명할 수 있는 근거가 잘 드러나도록 기술) (700~1000자 10단락 이내) (700~1000자 10단락 이내)*

생산 공정 최적화를 위한 데이터 분석 및 시뮬레이션 기술에 전문성을 키웠습니다. 특히 반도체 제조 공정에서의 공정 최적화와 대용량 데이터 기반 성능 개선에 집중했습니다. Silvaco 시뮬레이션 도구 마스터부터 시작하여 FinFET 공정에서 온도, 물질량, 공정 횟수 등 핵심 변수들의 상관관계를 분석하며 공정 파라미터 최적화 방법론을 체계적으로 학습했습니다. 그 결과, FinFET 프로젝트에서 thermal budget 감소(1000°C→800°C), 물질 사용량 30% 절약, 공정 단계 5회 감소를 달성했습니다. 이후 BigQuery, SQL, Python을 익혀 대용량 데이터 처리 역량을 보강했습니다.

6개월간 매일 8시간 이상 집중 학습하며 이론 학습(30%) + 실습 프로젝트(50%) + 자격증 취득(20%) 비율로 구성했습니다. 특히 실제 데이터를 활용한 프로젝트 중심 학습을 통해 이론과 실무의 연결고리를 확립했습니다. 배달앱 분석에서는 52,823명 사용자 데이터로 휴면율 문제 해결 방안을 도출했으며, 신규 유저의 90%가 4주 내 이탈, 리텐션율이 8%이라는 결과를 인지하고, 최우선적으로 고객들이 떠나지않도록 온보딩 프로세스 개선 및 휴면 유저 리액티베이션 캠페인 등의 개선 방안을 제시했습니다.

또한 SQLD, 컴퓨터활용능력 1급, AICE Associate 자격증 취득으로 제가 가진 역량을 검증받았습니다. 특히 AICE Associate는 머신러닝 기반 공정 최적화 역량을 객관적으로 입증하는 지표입니다. 데이터 분석 부트캠프를 통해 실무진들과 지속적으로 교류하며 최신 기술 동향과 현장 적용 사례를 공유하고 있습니다. 이를 통해 이론적 지식을 실무 중심으로 발전시키고 있으며, 이 전문성을 바탕으로 MES 데이터 분석, 공정 가동율 최적화, Lead Time 단축, Recipe 개선 등 귀사 생산관리/생산기술 업무에 직접 기여할 수 있습니다.

데이터 분석 부트캠프에서 서로 다른 전공 배경을 가진 4명의 팀원(경영학과, 컴퓨터공학과, 전자공학과인 저, 심리학과)과 함께 배달앱 사용자 행동 분석 프로젝트를 진행했습니다. 저는 데이터 전처리 및 기술적 구현 담당을 맡았습니다. 170만 건의 방대한 데이터를 5주간에 걸쳐 분석하면서 사용자 세분화, 비즈니스 인사이트 도출, 마케팅 전략 수립까지 포괄하는 종합적 프로젝트였기에 각 전공별 전문성이 필요한 영역들(통계 분석, 사용자 심리 분석, 비즈니스 모델 이해)이 명확히 구분되어 협업이 필수적이었습니다.

팀 전체가 "실제 비즈니스에 적용 가능한 데이터 기반 개선 방안 도출"이라는 공동 목표를 설정했습니다. BigQuery 환경 구축과 SQL 쿼리 최적화는 제가 담당했고, 통계학과 팀원은 코호트 분석, 경영학과 팀원은 비즈니스 모델 분석, 심리학과 팀원은 사용자 행동 패턴 해석을 맡았습니다. 컴공과 팀원과는 데이터 시각화 및 대시보드 구축을 협업했습니다. 

초기에 "휴면 유저는 포기하고 신규 유저 확보에 집중해야 한다"는 의견과 "휴면 유저 리액티베이션이 더 효율적"이라는 의견이 충돌했으며, 데이터로 가설을 검증하자고 제안했습니다. 분석 결과, 52,823명의 전체 사용자 중 29,995명이 휴면임을 밝혀내고,휴면 유저 결제율(18.65%) vs 신규 유저 결제율(13.38%) 비교 분석을 통해 휴면 유저의 가치를 객관적으로 입증했습니다. 이를 바탕으로 "데이터 기반 의사결정"이라는 공통 원칙을 확립하며 팀의 방향성을 통일했습니다. 최종적으로 1주차 리텐션 8%→20% 개선, 휴면율 57%→40% 감소라는 구체적 목표와 6가지 실행 방안을 완성했습니다. 저는 핵심 데이터 분석과 기술적 검증을 담당하며, 배달앱 전체 분석 리포트 제작 및 BigQuery SQL 코드 등 성과의 약 30%에 기여했다고 생각합니다. 경험을 통해 다양한 전문성을 가진 구성원들과의 협업 능력과 데이터 기반 갈등 해결 방법을 체득했습니다.
